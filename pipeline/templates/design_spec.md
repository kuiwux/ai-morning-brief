# 硅谷AI晨报 · 公众号排版设计规范 v1.0

> **设计团队**：秋山梦 UI/UX  
> **风格定义**：暖色报纸风（Warm Newspaper Style）  
> **适配平台**：微信公众号图文消息  
> **核心约束**：纯内联样式 · Table 布局 · 宽 ≤ 600px · 无 JS · 无外链 CSS  

---

## 一、设计理念

### 1.1 风格定位

「硅谷AI晨报」是一份科技日报，品牌调性要求**专业、可信、有温度**。设计以传统报纸排版为灵感，融入暖色系配色：

- **传统报纸的秩序感**：双线刊头、板块标签、层级分明的标题体系
- **暖色调的情感温度**：深棕色替代纯黑，米黄/奶油色替代纯白
- **数字阅读的呼吸感**：充裕的行距、段距，避免信息密度过高造成的压迫感

### 1.2 视觉层级

```
┌──────────────────────────────────────────┐
│  L1: 封面图（首屏冲击）                    │
├──────────────────────────────────────────┤
│  L2: 元数据行（日期/来源数/时长）           │
├──────────────────────────────────────────┤
│  L3: 报纸刊头（品牌锚点）                  │
├──────────────────────────────────────────┤
│  L4: 今日头条（最重要内容，暖色卡片突出）     │
├──────────────────────────────────────────┤
│  L5: AI 快讯列表（按主题分组，常规信息流）   │
├──────────────────────────────────────────┤
│  L6: ELI5 区块（特殊视觉，左色条+浅底）     │
├──────────────────────────────────────────┤
│  L7: 编辑精选专题入口（渐变卡片+CTA）        │
├──────────────────────────────────────────┤
│  L8: 数据概览 + 页脚引导                   │
└──────────────────────────────────────────┘
```

## 四、间距规范

### 4.1 外边距系统

```
内边距体系（基于 4px 基准）：

  4px   - 紧凑间距（元素内微调）
  8px   - 小间距（标题与正文之间）
  12px  - 中等间距（卡片内元素之间）
  16px  - 常规间距（卡片 padding）
  20px  - 大间距（模块之间，两侧留白）
  24px  - 特大间距（封面区域边缘）
```

### 4.2 模块间距

```
  [封 面 图]          padding: 0
       ↓               margin-bottom: 0
  [元数据行]           padding: 16px 20px 10px 20px
       ↓               border-top
  [报纸刊头]           padding: 18px 20px 14px 20px
       ↓               双线装饰
  [今日头条]           padding: 20px
       ↓               装饰分割线
  [AI 快讯]            padding: 0 20px 20px 20px
       ↓               border-top
  [今日 ELI5]          padding: 18px 20px 8px 20px
       ↓               border-top
  [编辑精选专题]        padding: 18px 20px
       ↓               border-top
  [数据一览]           padding: 16px 20px
       ↓               无分割
  [页脚引导]           padding: 0 20px 30px 20px
```

### 4.3 分割线规范

| 类型 | 样式 | 用途 |
|------|------|------|
| 双线（刊头） | 1px `#C8A882` + 3px `#8B6914`，间距 3px | 报纸刊头下方，品牌标志 |
| 粗实线 | 2px `#C8A882` | 大板块之间分隔 |
| 细实线 | 1px `#E0D0B8` | 次级分隔 |
| 虚线 | 1px dashed `#E0D0B8` | 主题分组之间 |
| Emoji 装饰 | `✦ ✦ ✦`，12px，`#C8A882`，6px 字间距 | 头条与快讯之间 |
| 粗底栏 | 3px `#8B6914` | 页脚上方，与刊头呼应 |

## 五、组件规范

### 5.1 组件总览

| # | 组件名称 | 文件名/Class | 对应数据 |
|---|---------|-------------|---------|
| 1 | 元数据行 | `metadata-row` | `meta.date`, `meta.source_count` |
| 2 | 报纸刊头 | `masthead` | 品牌常量 |
| 3 | 板块标签 | `section-label` | 板块名称常量 |
| 4 | 头条卡片 | `headline-card` | `daily_summary.headline` |
| 5 | 简讯条目 | `brief-item` | `daily_summary.sections[].items[]` |
| 6 | 主题分组 | `topic-group` | `daily_summary.sections[]` |
| 7 | ELI5 块 | `eli5-block` | `eli5.items[]` |
| 8 | 来源标注行 | `source-line` | `item.sources[]` |
| 9 | 专题入口卡片 | `topic-entry-card` | `curated_topics[]` |
| 10 | 分割线 | `divider` | 无 |
| 11 | 数据概览面板 | `stats-panel` | `meta.article_count` |
| 12 | 页脚关注引导 | `footer-cta` | 品牌常量 |

### 5.2 关键组件设计要点

#### 头条卡片
- 背景 `#FFF7ED`，1px `#E8D5B7` 边框，4px 圆角
- 内边距 16px
- 标题使用衬线字体 18px，摘要 15px，来源 12px
- 与周围内容有明确视觉区分

#### ELI5 块
- 背景 `#FFF9F0`，**左侧 4px 暖金色竖条** `#E8A650`
- 右侧 4px 圆角（`border-radius: 0 4px 4px 0`）
- 三条 ELI5 之间间距 14px
- 「💡 为什么重要」使用 `#C8862A` 琥珀色，加粗
- 不设底部圆角以适配多条连续排列

#### 简讯条目
- `▸` 三角符号前置，颜色 `#C8A882`
- 标题 14px/600，摘要 13px，来源 11px
- 条目间距 10-12px

#### 专题入口卡片
- 渐变背景 `linear-gradient(135deg, #FFF7ED, #FFF0E0)`
- 1px `#E0C8A0` 边框，6px 圆角
- 可选封面缩略图（560×200）
- CTA 按钮：深棕底 `#8B4513`，白色文字，居中

## 六、微信兼容性清单

### 6.1 ✅ 已知兼容

| 特性 | 兼容状态 |
|------|---------|
| `table` 布局 | ✅ 完全支持 |
| 内联 `style` | ✅ 完全支持 |
| `border-radius` | ✅ 支持 |
| `letter-spacing` | ✅ 支持 |
| `<a>` 链接 | ✅ 认证公众号支持外链 |
| Emoji | ✅ 支持 |
| `text-align: justify` | ✅ 支持 |
| 基本 `border` 样式 | ✅ 支持 |
| `padding` / `margin` | ✅ 支持 |
| `font-family` 指定 | ✅ 支持 |

### 6.2 ❌ 已知不兼容

| 特性 | 状态 | 替代方案 |
|------|------|---------|
| `flexbox` | ❌ 不支持 | 使用 table 布局 |
| `grid` | ❌ 不支持 | 使用 table 布局 |
| `position: fixed/sticky` | ❌ 不支持 | 不使用 |
| `background-image` (某些场景) | ⚠️ 部分支持 | 使用纯色背景 |
| `@font-face` | ❌ 不支持 | 使用系统字体栈 |
| `<style>` 标签 | ⚠️ 会被过滤 | 仅用内联 `style` 属性 |
| JavaScript | ❌ 被完全过滤 | 不使用任何 JS |
| 外部 CSS 文件 | ❌ 不支持 | 所有样式内联 |
| `max-width` | ⚠️ 部分支持 | 用 `width="600"` 兜底 |
| SVG | ⚠️ 部分支持 | 装饰用 CSS border 替代 |

### 6.3 代码规范

- ✅ 所有样式写在 `style=""` 属性中
- ✅ 使用 `<table>` 实现多列布局
- ✅ `<img>` 标签始终设置 `width` / `height` 属性
- ✅ `<a>` 链接设置 `text-decoration:none`
- ✅ 不使用 `margin-top` / `margin-bottom` (用 `padding` 替代，微信兼容性更好)
- ❌ 不使用 `background-image` / `linear-gradient` (仅在封面 CSS 描述中使用，实际模板中避免)

## 七、模板变量映射

Jinja2 模板中可用变量与 JSON Schema 的对应关系：

```
{{ meta.date }}                    → 元数据·日期
{{ meta.source_count }}            → 元数据·来源数
{{ meta.article_count }}           → 数据面板·文章数
{{ daily_summary.headline }}       → 今日头条·标题
{{ daily_summary.sections }}       → AI 快讯·主题分组
{{ section.topic }}                → 分组标题
{{ item.headline }}                → 简讯标题
{{ item.summary }}                 → 简讯摘要
{{ item.sources }}                 → 来源列表
{{ item.is_exclusive }}            → 是否独家
{{ eli5.items }}                   → ELI5 条目列表
{{ eli5_item.eli5_text }}          → ELI5 正文
{{ eli5_item.why_matters }}        → 为什么重要
{{ curated_topics }}               → 专题列表
{{ topic.title }}                  → 专题标题
{{ topic.cover_story }}            → 专题摘要
```
